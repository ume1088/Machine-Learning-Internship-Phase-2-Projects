# AI/ML Engineering Internship – Phase 2 Tasks

This repository contains the **Phase 2 projects** completed as part of the AI/ML Engineering internship. The projects focus on practical applications of machine learning and natural language processing using real-world datasets. Each project has its own folder with detailed implementation and individual README files.

---

## Project 1: End-to-End ML Pipeline for Customer Churn Prediction

**Dataset:** Telco Churn Dataset  
**Objective:**  
The objective of this project is to build a complete **machine learning pipeline** for predicting customer churn in a telecom company. The pipeline includes:

- Data preprocessing and feature engineering  
- Handling missing values and categorical encoding  
- Model training and evaluation using various algorithms  
- Performance metrics analysis (Accuracy, F1 Score, ROC-AUC)  
- Visualization of important features and results  

This project demonstrates **end-to-end ML workflow** from raw data to actionable insights for business decision-making.

---

## Project 2: News Classifier with BERT Model

**Dataset:** AG News Dataset  
**Objective:**  
This project focuses on **text classification using BERT**. The main goals include:

- Fine-tuning a pre-trained BERT model for multi-class news classification  
- Implementing zero-shot and few-shot learning strategies  
- Evaluating model performance using accuracy and F1-score  
- Automating news categorization into predefined classes: Business, Science/Tech, Sports, and World  

The project highlights the use of **transformer-based language models for NLP tasks** and demonstrates practical text classification workflows.

---

## Project 3: Auto Tagging Support Tickets Using LLM

**Dataset:** Banking77 Dataset  
**Objective:**  
This project implements an **automatic support ticket tagging system** using a Large Language Model (LLM). The goals include:

- Categorizing customer support tickets into relevant labels automatically  
- Implementing both **zero-shot** and **few-shot classification** approaches  
- Generating the **top three most probable tags** for each ticket  
- Comparing zero-shot and few-shot predictions for improved contextual understanding  

This project demonstrates **LLM-based text classification for real-world customer support automation**, reducing manual workload and improving ticket routing efficiency.
